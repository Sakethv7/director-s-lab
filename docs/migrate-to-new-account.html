Migrate to a New Google Account
Step-by-step guide to re-commission all GCP resources and claim free credits on a new account
Project: Director's Lab Prepared by: Adithya Date: March 2026

What You Are Moving

ServiceWhat it doesBillable?
Cloud RunHosts the FastAPI backend (directors-lab-api)Yes — per request
Firebase HostingHosts the React frontendFree tier is generous

Phase 1 — New Google Account & GCP Project

1
Create a new Google Account
If you already have an account with unused free trial credits, skip to Step 2
  1. Go to accounts.google.com → Create account
  2. Use a new email address (e.g. a fresh Gmail)
  3. Complete account setup
2
Create a new GCP Project and activate free trial
You get $300 USD in free credits valid for 90 days
  1. Go to console.cloud.google.com and sign in with the new account
  2. Click Activate Free Trial — you will need to enter a credit card (you will NOT be charged unless you manually upgrade)
  3. Click Select a project → New Project
  4. Name it: Director's Lab — GCP will auto-assign a Project ID like directors-lab-XXXXXX
  5. Note the exact Project ID shown below the name field — you will paste it into every command below
ℹ
The Project ID is auto-generated (e.g. directors-lab-482910) and is different from the name. Write it down — you'll need it in Steps 3, 8, 9, 10, and 11.
3
Enable required APIs
Run in Google Cloud SDK Shell
gcloud config set project gemini-live-agent-challenge-26

gcloud services enable \
  run.googleapis.com \
  cloudbuild.googleapis.com \
  artifactregistry.googleapis.com \
  aiplatform.googleapis.com \
  firestore.googleapis.com \
  storage.googleapis.com \
  texttospeech.googleapis.com

Phase 2 — Firebase Setup

4
Link Firebase to your new GCP project
Firebase Hosting needs to be connected to the project
  1. Go to console.firebase.google.com
  2. Click Add project → select your existing GCP project from the list
  3. Follow the setup wizard (you can disable Google Analytics if you don't need it)
  4. Once linked, run from your project root directory: 
    firebase login
firebase use --add
    Select your new GCP project when prompted.
5
Create Firestore database
The backend stores every scene here
  1. In Firebase Console → go to Firestore Database
  2. Click Create database
  3. Choose Native mode
  4. Select region: us-central1 (must match Cloud Run region)
  5. Start in production mode (the firestore.rules file already handles permissions)
ℹ
The firestore.rules file in the project root allows the backend to write and everyone to read. Deploy these rules with: firebase deploy --only firestore:rules

Phase 3 — Cloud Storage Bucket

6
Create and configure the GCS bucket
Run in Google Cloud SDK Shell

Use the same bucket name as before: gemini-hackathon-2026-bucket
Note: This only works if you have already deleted the old bucket from the previous account. If not, delete it first with gsutil rm -r gs://gemini-hackathon-2026-bucket on the old account.

# Create the bucket
gsutil mb -p gemini-live-agent-challenge-26 -l us-central1 gs://gemini-hackathon-2026-bucket

# Enable uniform bucket-level access (required)
gsutil uniformbucketlevelaccess set on gs://gemini-hackathon-2026-bucket

# Make all objects publicly readable (so image/audio/video URLs work)
gsutil iam ch allUsers:objectViewer gs://gemini-hackathon-2026-bucket

Phase 4 — Service Account (for Local Dev)

7
Create a service account with required roles
Only needed for local development. Cloud Run uses its default service account automatically.
  1. In GCP Console → IAM & Admin → Service Accounts
  2. Click Create Service Account
  3. Name it: directors-lab-sa
  4. Grant it these roles:
    • Vertex AI User — for Imagen 3, Lyria, Veo 3.1
    • Cloud Datastore User — for Firestore read/write
    • Storage Object Admin — for GCS upload and public URL access
  5. Click Done
  6. Click on the service account → Keys tab → Add Key → JSON
  7. Download the JSON file and save it to C:\Users\Adithya\Downloads\ — same place as before
⚠
Never commit this JSON key file to Git. The .gitignore already excludes it via the pattern *credentials*.json and service-account*.json, but double-check before any commit.

Phase 5 — Update Environment Variables

8
Update backend/.env (local development)
This file is gitignored — safe to edit freely

Open backend/.env. Only the 3 lines marked ← CHANGE need updating. Everything else is identical to before.

GEMINI_API_KEY=

GEMINI_MODEL=gemini-2.5-flash

GOOGLE_APPLICATION_CREDENTIALS="C:\Users\Adithya\Downloads\YOUR_NEW_KEY_FILE.json"  # ← CHANGE: new JSON key filename

GOOGLE_CLOUD_PROJECT=gemini-live-agent-challenge-26          # same as before
GOOGLE_CLOUD_REGION=us-central1

CORS_ORIGINS=https://gemini-live-agent-challenge-26.web.app,https://gemini-live-agent-challenge-26.firebaseapp.com  # same as before

FIRESTORE_COLLECTION=scenes

GCS_BUCKET_NAME=gemini-hackathon-2026-bucket
ℹ
Only 1 thing changes: the GOOGLE_APPLICATION_CREDENTIALS filename — because the new account generates a new JSON key file. Every other value is identical to the old account.
9
Update backend/env-vars.yaml (Cloud Run deployment)
Used during gcloud deploy — also gitignored

Open backend/env-vars.yaml. Only the 2 lines marked ← CHANGE need updating.

GEMINI_API_KEY: ""
GEMINI_MODEL: "gemini-2.5-flash"
GOOGLE_CLOUD_PROJECT: "gemini-live-agent-challenge-26"                                                    # same as before
GOOGLE_CLOUD_REGION: "us-central1"
GCS_BUCKET_NAME: "gemini-hackathon-2026-bucket"
FIRESTORE_COLLECTION: "scenes"
CORS_ORIGINS: "https://gemini-live-agent-challenge-26.web.app,https://gemini-live-agent-challenge-26.firebaseapp.com"   # same as before
ℹ
Note: env-vars.yaml does not have a GOOGLE_APPLICATION_CREDENTIALS line — Cloud Run uses its attached service account automatically, no JSON key needed.

Phase 6 — Deploy Backend

10
Authenticate gcloud with the new account
Run in Google Cloud SDK Shell
gcloud auth login                                  # sign in with project.account.for.building@gmail.com
gcloud config set project gemini-live-agent-challenge-26
gcloud auth application-default login
11
Deploy backend to Cloud Run
Google Cloud Build builds the Docker image automatically — no local Docker needed
gcloud run deploy directors-lab-api \
  --source "d:\Data Engineering\director-s-lab\backend" \
  --region us-central1 \
  --allow-unauthenticated \
  --memory 2Gi \
  --timeout 300 \
  --env-vars-file "d:\Data Engineering\director-s-lab\backend\env-vars.yaml"

After deployment, note the Cloud Run URL printed at the end. It looks like:

https://directors-lab-api-XXXXXXXXXX-uc.a.run.app
12
Confirm the correct Cloud Run URL
gcloud run services describe directors-lab-api \
  --region us-central1 \
  --format="value(status.url)"

Copy this URL — you need it in the next step.

Phase 7 — Deploy Frontend

13
Update frontend/.env.local with the new Cloud Run URL
This bypasses Firebase Hosting's 60-second proxy timeout for long AI requests

Open frontend/.env.local and replace the URL:

The old URL was: https://directors-lab-api-y4fm425eyq-uc.a.run.app
Replace it with the new URL from Step 12.

VITE_API_URL=https://directors-lab-api-XXXXXXXXXX-uc.a.run.app  # ← paste new URL here
14
Update firebase.json with the new Cloud Run service URL
The rewrite rule must point to the new project's Cloud Run service

Open firebase.json in the project root. Find the rewrites section and confirm the run block has:

{
  "source": "/api/**",
  "run": {
    "serviceId": "directors-lab-api",
    "region": "us-central1"
  }
}

The serviceId stays the same (directors-lab-api) — Firebase resolves it relative to whichever project is active.

15
Build and deploy the frontend
Run from the project root in any terminal
cd frontend
npm run build
cd ..
firebase deploy --only hosting

Firebase Hosting URL — same as before:

https://gemini-live-agent-challenge-26.web.app

Phase 8 — Test Everything

16
Run these checks in order
  • Health check passes: https://directors-lab-api-XXXXXXXXXX-uc.a.run.app/health returns {"status":"ok"}
  • Frontend loads at https://gemini-live-agent-challenge-26.web.app
  • Pitch a scene → clarifying question appears (Gemini is working)
  • Answer the question → storyboard frame generates with an image (Imagen 3 is working)
  • Video clip appears on the storyboard (Veo 3.1 is working)
  • Scene is saved — refresh the page, scene still loads (Firestore is working)
  • Director's Note revision regenerates the video (full loop working)

Phase 9 — Decommission Old Account (Optional)

17
Stop billing on the old GCP project
Do this only after confirming the new account works

Run in Google Cloud SDK Shell (authenticated as the OLD account):

# Delete the old Cloud Run service (stops compute billing)
gcloud run services delete directors-lab-api \
  --region us-central1 \
  --project gemini-live-agent-challenge-26

# Delete the old GCS bucket
gsutil rm -r gs://gemini-hackathon-2026-bucket

# Delete the old Artifact Registry images
gcloud artifacts repositories delete cloud-run-source-deploy \
  --location us-central1 \
  --project gemini-live-agent-challenge-26

Or for a full nuclear delete of the old project:

gcloud projects delete gemini-live-agent-challenge-26
⚠
Project deletion takes 30 days to become permanent. You can restore it within that window at console.cloud.google.com/cloud-resource-manager if needed.
