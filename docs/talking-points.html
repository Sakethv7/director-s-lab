0 — Introductions

Both Who We Are
[ camera on both speakers — app visible in background ]
Saketh "Hey, I'm Saketh — I worked on the product, the frontend, and the overall user experience."
Adi "And I'm Adi — I built the backend, the AI pipeline, and the Google Cloud infrastructure."
Both What We Built
[ switch to the app — landing page visible ]
Saketh "Together we built Director's Lab."
Adi "A multimodal AI directing agent that takes a scene you describe and turns it into a full storyboard — images, music, voiced dialogue, and a cinematic video clip."
Saketh "All powered by Gemini, Vertex AI, and Google Cloud. Let us show you how it works."

1 — Opening

Saketh Hook & Introduction
[ landing page visible, nothing typed yet ]

"Every director has a scene in their head. The light, the silence, the moment someone doesn't look back. The hard part is getting it out before the feeling dies."

"Director's Lab fixes that. You pitch the moment — by text, voice, or a reference image. The AI takes it from there."

[ point to the three input options on screen ]

"Built for the Gemini Live Agent Challenge. Everything runs on Google Cloud."

2 — The Problem

Saketh Why Existing Tools Fall Short
[ still on landing page ]

"Every AI tool today gives you one output. One prompt, one image, done. But directors don't think in single frames — they think in arcs."

"Director's Lab builds a 4-beat story structure — Establish, Escalate, Tension, Resolve — and every output, image, audio, video, dialogue, all stays in the same emotional tone."

[ pitch a scene, wait for clarifying question to appear ]

"And before it generates anything — it asks one question. Not to stall. To lock in the feeling before the pipeline runs."

3 — The Tech

Adi Five Models, One Pipeline
[ answer the question, generation begins — stay on loading state ]

"Five AI models are running at the same time right now on Vertex AI."

"Gemini 2.5 Flash is writing the story arc. Imagen 3 is painting all four frames. Lyria is generating an ambient score. Gemini TTS is voicing every line with gender-matched voices. And Veo 3.1 is rendering a cinematic video clip."

[ scene loads — pause for a beat ]

"All five. Same emotional brief. Same register."

[ point to the beat map on the sidebar ]

"This beat map — tension, longing, resolve — is what keeps them aligned. It's the shared language every model works from."

4 — What Makes It Different

Adi The Human-in-the-Loop Revision Loop
[ type a director's note: "make it colder, less dialogue" ]

"The human stays in control. The AI never quietly decides what to change."

[ click preview revision — the HITL modal appears ]

"Before a single image regenerates, Gemini shows you what it wants to change — which panels, why, and how the beat map shifts."

"You see exactly what's going to change before anything regenerates — the beat map shift and why. Then you confirm and it applies the revision."

[ click apply ]

"No surprises, no lost work. You're the director — the AI is just the crew."

5 — The Build

Adi The Hard Parts
[ switch to GCP console — Cloud Run → directors-lab-api → Logs ]

"Three things were genuinely tricky to build."

"First — Veo 3.1 doesn't return a video instantly. You submit the job and poll until it's done. We built a polling loop with a timeout and a fallback so it never crashes the whole pipeline."

[ scroll to show the wall of Vertex AI POST requests in logs ]

"Second — five AI calls running at the same time without any of them blocking the others. We used asyncio and background executors to keep the event loop free."

"Third — merging the Lyria score, the TTS voice, and the Veo clip into one 15-second video using ffmpeg, all inside a Cloud Run container with nothing installed locally."

[ zoom into the log line: "Panel 1: 15s video merged (ambient=True voice=True)" ]

6 — What We Learned

Adi Learnings
[ switch back to the finished scene with video visible ]

"The minute-long wait for Veo is actually a good thing — it proves nothing is pre-generated. That's a real model rendering in real time."

"Vertex AI's long-running operations pattern makes sense once you work with it — it's the right way to handle jobs that take time, like video generation."

"Cloud Run was the perfect fit. It handled the bursty AI workload, scaled down to zero between requests, and we never had to think about servers."

Saketh The Product Insight

"The biggest thing we learned was about the product itself. The HITL loop changed how we thought about what we were building."

"This isn't about replacing the director. It's about giving them a crew that actually listens."

7 — Closing

Both Sign-off

Saketh "Director's Lab."
Adi "Gemini writes it."
Saketh "Imagen paints it."
Adi "Lyria scores it."
Saketh "Veo renders it."
Adi "You direct it."
[ one beat of silence ]
Both "Shoot, cut, and direct at 24fps."
